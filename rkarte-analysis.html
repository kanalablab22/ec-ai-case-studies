R-Karteアクセス分析
自動蓄積 & AI即答

楽天R-Karteの参照元・検索キーワードデータを毎朝自動で取得し、ローカルDBに蓄積。 Claude Codeに聞くだけで流入経路やキーワードランキングが即座に返ってくる仕組みを実現しました。

カテゴリ データ蓄積・AI連携
技術スタック Python / AppleScript / SQLite / MCP
取得頻度 毎朝9:15 自動実行
対応店舗 Shop A / Shop B（2店舗）
課題と解決策

😰 Before（課題）

  • アクセス分析を見るたびにRMSにログインしてR-Karteを開く必要があった
  • 参照元や検索キーワードの推移を見たくても、過去データを遡る操作が面倒
  • R-KarteのデータはAPIが公開されておらず、自動取得の手段がなかった

🎉 After（解決）

  • Claude Codeに「昨日のキーワード教えて」と聞くだけで即回答
  • 毎朝自動で2店舗分のデータを取得、日々蓄積されるので推移分析も可能
  • AppleScriptでChromeの既存セッションを活用し、ログイン不要でデータ取得

技術的なポイント

🔓 ログイン不要の仕組み

R-KarteにはAPIが公開されていないため、AppleScriptでChromeの既存セッションを活用し、ログイン不要でデータを取得する仕組みを実現。発想の転換でシンプルに解決しました。

AppleScript + osascript

📊 隠れたJSON APIを発見

R-Karteの画面をDevToolsで分析し、内部で使われているJSON APIを特定。HTMLスクレイピング不要で、構造化されたデータを直接取得できるようになりました。

fetch() API

🔌 MCP（Model Context Protocol）

蓄積したデータをMCPサーバー経由でClaude Codeに接続。自然言語で質問するだけで、SQLiteのデータを検索・集計して回答する仕組みです。

FastMCP + stdio

🏪 複数店舗の同時対応

Chromeの別プロファイルで2店舗に同時ログイン。AppleScriptが各ウィンドウのR-Karteタブを自動検出し、両店舗のデータを一度に取得します。
